# 방명록 설정 가이드 (Supabase 연동)

방명록은 모든 방문자에게 공유되므로, 글을 저장할 무료 데이터베이스(Supabase)가 필요합니다.
아래 순서를 그대로 따라 하시면 5분 안에 완료됩니다.

## 1단계. Supabase 가입 및 프로젝트 생성
1. https://supabase.com 접속 → **Start your project** 클릭 후 무료로 가입 (GitHub 계정으로 가입 가능)
2. **New Project** 클릭
   - Name: 원하는 이름 (예: `choi-professor-guestbook`)
   - Database Password: 임의로 설정 (기억해두지 않아도 됨)
   - Region: `Northeast Asia (Seoul)` 선택 권장
3. 프로젝트 생성까지 1~2분 대기

## 2단계. 방명록 테이블 만들기
1. 왼쪽 메뉴에서 **SQL Editor** 클릭
2. **New query** 클릭 후 아래 코드를 붙여넣고 **Run** 클릭

```sql
create table guestbook (
  id uuid primary key default gen_random_uuid(),
  name text not null,
  message text not null,
  created_at timestamptz not null default now()
);

alter table guestbook enable row level security;

-- 누구나 글을 읽을 수 있도록 허용
create policy "Public read access"
on guestbook for select
to anon
using (true);

-- 누구나 글을 남길 수 있도록 허용 (수정/삭제는 불가)
create policy "Public insert access"
on guestbook for insert
to anon
with check (true);
```

## 3단계. API 키 복사하기
1. 왼쪽 메뉴 하단 **Project Settings**(⚙️) → **API** 클릭
2. 다음 두 값을 복사해 둡니다.
   - **Project URL** (예: `https://abcdefgh.supabase.co`)
   - **Project API keys** 항목의 **anon public** 키

## 4단계. index.html 파일에 붙여넣기
`index.html` 파일에서 아래 부분을 찾아 값을 교체합니다.

```javascript
const SUPABASE_URL = 'https://YOUR_PROJECT_ID.supabase.co';   // ← 2단계에서 복사한 Project URL
const SUPABASE_ANON_KEY = 'YOUR_ANON_PUBLIC_KEY';              // ← 2단계에서 복사한 anon public 키
```

저장 후 웹사이트에 다시 올리면(재배포) 방명록이 정상적으로 작동합니다.

## 참고사항
- **비용**: Supabase 무료 플랜으로도 개인 홈페이지 방명록 용도로는 충분합니다.
- **스팸 방지**: 기본적인 봇 차단(honeypot) 로직이 포함되어 있지만, 악성 게시글이 많다면 Supabase 대시보드 → **Table Editor** → `guestbook` 테이블에서 직접 삭제할 수 있습니다.
- **글 삭제 권한 부여하고 싶다면**: 현재는 누구나 삭제할 수 없도록 되어 있습니다(관리자만 대시보드에서 삭제 가능). 필요 시 말씀해 주시면 관리자 전용 삭제 기능도 추가해 드릴 수 있습니다.
